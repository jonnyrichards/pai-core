## WORKSHOP OVERVIEW *(10 min)*

We're here because three things are converging: **Routing**, **Unified Sessions**, and **MSS**. Two big milestones are forcing our hand — **Coach on the Home Page** and **Unified Session History**. To deliver either well, we need answers to four questions we've been dancing around:

1. Should Coach appear on every product/screen?
2. Should any Coach module be accessible from any other module?
3. Should every past session be available from every module?
4. Should the Primary pane always match the active Coach conversation — and if not, when should it?

We're not going to answer these abstractly. We'll answer them by working through real user journeys first, then distilling what we find into principles.

Two tracks worth naming upfront:
- **Critical Path** — what we're actually building
- **Future State** — what we'd do if platform constraints disappeared

Ideally we're thinking about both simultaneously. When you're working, flag if something is a current constraint vs. a genuine design choice.

> **Any questions before we get going?**

## SECTION 1 — User Journeys *(45 min)*

### Brief *(5–10 min)*

Shortly we'll split into teams. Each team gets a persona — one LoS, one manager, one HR admin.

**Two tasks:**

**Task 1 — Map one or two user flows.** Real sequences of what that person does in the product. Use these scenarios to inform your flow (pick at least 3):
- User navigates to another page on the platform
- User opens an old session from session history
- User (manager) revisits an incomplete MR page and Coach auto-opens
- User navigates back
- User (LoS) lands on home page and asks a general coaching question (not MSS-specific)
- User in one Coach context (e.g. MR page, survey report) asks a question that another Coach module is better equipped to answer

**Task 2 — Decide how Coach should behave UX-wise at each step.**
- Does it persist?
- Does it change modules?
- Does a new Coach session start?

Don't worry about abstracting out — just describe the user flow clearly.

> **30 minutes. Meet back at [time].**

## SECTION 2 — Unified Session Principles *(40 min)*

### Brief *(5 min)*

New teams! We're mixing it up into more functional groups:
- **Team 1:** Engineering
- **Team 2:** Product, Design, Content

Each team drafts **up to 5 principles** that describe how Coach behaves across the platform.

**The constraint:** your principles should implicitly answer the four questions from the overview — you shouldn't need to answer them directly. A good set of principles makes the answers obvious.

**Two parts:**
- **Part 1 *(25 min)*:** Draft your 5 principles
- **Part 2 *(10 min)*:** Swap notes — synthesise the other team's principles into yours. Bonus points for a ranked list.
