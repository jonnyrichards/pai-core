- 2026-06-16 [work, enablement, evals]: **Coach Evals Process** (source: Kristina, PSX researcher). **Step 1 — Scenario generation** (Kristina): Input: new agent / system prompt. Kristina runs a Claude skill that determines which questions are needed (performance evals for this prompt, guardrails evals, global evals) and outputs a structured sheet of scenarios — e.g. "Does the Coach hallucinate?" / "Does it treat a behaviour mentioned in only one source as a wider behaviour?" Sheet is organised by Coach module + global questions. Output: scenarios sheet (input to Step 2). **Step 2 — Conversation simulation** (Eng): Input: scenarios sheet. Eng takes each scenario and generates a conversation with Coach. Output: conversation log. **Step 3 — Judge evaluation** (Eng): Input: conversation log. A judge LLM evaluates each response against the expected behaviour. Output: one results sheet per eval run (e.g. for Manager Review, Voice). **Step 4 — QA** (Kristina): Input: judge results sheet. Kristina manually scans to verify the judge is behaving correctly. Output: sign-off (or flag for re-run). **Step 5 — Summary write-up** (Eng / DS): Input: results sheet. A Confluence summary document is written (e.g. "Evaluation Run — Langgraph MR agent"), added to the evals hub, and (ideally) shared with the PM. **Related:** ##proj_coach_evals (Slack); Coach Evaluation Hub: https://cultureamp.atlassian.net/wiki/spaces/coach/pages/5859737650/Coach+Evaluation+Hub
